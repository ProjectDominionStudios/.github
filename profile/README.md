# Project Dominion Studios

**Building technology with purpose.**

Project Dominion Studios is an independent, AI-native technology and innovation studio focused on designing, building, and maintaining practical software, intelligent automation, developer tools, business applications, cloud and infrastructure solutions, research projects, and interactive experiences.

Game development remains an important area of exploration, but it is one part of a broader mission: creating thoughtful technology that solves real problems, strengthens human capability, and improves the systems and processes around it.

## Mission

Build useful, maintainable technology through curiosity, disciplined engineering, continuous learning, responsible AI collaboration, and clear documentation.

## Vision

Create a sustainable studio where software, operational knowledge, engineering standards, and professional growth reinforce one another—and where every completed project leaves behind better tools, better processes, and stronger capabilities.

## What we build

- **Business and productivity applications** — practical tools that simplify work, organize information, and reduce repetitive effort.
- **AI and automation** — responsible AI-assisted workflows, integrations, documentation systems, and task automation.
- **Developer tools** — utilities, validators, generators, diagnostics, and engineering workflow improvements.
- **Cloud and infrastructure solutions** — secure, maintainable systems, deployment patterns, monitoring, and operational tooling.
- **Research and learning projects** — experiments, proofs of concept, technical studies, and documented lessons.
- **Interactive experiences** — games, simulations, mods, narrative systems, and persistent online worlds.

## Current flagship project

### Galactic Dominion

A turn-oriented multiplayer strategy game focused on exploration, empire-building, diplomacy, fleet command, and a living galaxy shared with friends and rule-based AI empires.

- **Development codename:** Project Dominion
- **Current milestone:** Alpha 2 — First Contact
- **Current audience:** Private friends-and-family testing
- **Game domain:** `Project-Dominion.com`

Galactic Dominion is the studio's flagship interactive project, while additional studio initiatives explore productivity software, automation, developer tooling, cloud engineering, and AI-native development practices.

## How we work

Project Dominion Studios treats code, documentation, testing, architecture, learning, and operational resilience as first-class deliverables.

- Work begins with a clearly defined issue.
- One issue maps to one branch and one pull request.
- Changes remain small, reviewable, and testable.
- Decisions and lessons are documented while they are fresh.
- AI assists throughout the lifecycle; humans remain accountable.
- New ideas are captured without disrupting the active milestone.

## Studio principles

- Build with purpose.
- Finish before expanding.
- Keep progress small, meaningful, and sustainable.
- Document what you learn.
- Prefer evidence over assumptions.
- Adopt technology only when it solves a real problem.
- Automate repetitive work thoughtfully.
- Keep systems simple, secure, and maintainable.
- Protect contributor health, privacy, security, and trust.
- Leave every project better than you found it.
- Technology should serve people.

## Repository organization

The organization uses deliberate naming boundaries to keep studio operations separate from individual products and programs.

- **`.github` and `Studio.*`** repositories govern and operate Project Dominion Studios.
- **`ProjectDominion` and `ProjectDominion.*`** repositories build and operate the flagship Project Dominion program and Galactic Dominion product.
- **Founder and incubated project repositories** preserve private planning, learning, career development, experiments, and early-stage product work.

### Studio repositories

| Repository | Purpose |
|---|---|
| `.github` | Organization profile, contribution defaults, security guidance, and GitHub governance |
| `Studio.Management` | Studio operating standards, engineering practices, policies, and institutional history |
| `Studio.Design` | Studio identity, organization imagery, shared visual standards, social templates, and press assets |
| `Studio.Website` | `ProjectDominionStudios.com`, company information, project portfolio, press, and contact pages |
| `Studio.Infrastructure` | Studio domains, shared services, organization integrations, monitoring, backups, and operational policy |
| `Studio.Automation` | Studio publishing, newsletter, press, portfolio, and shared administrative automation |
